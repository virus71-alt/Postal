# On-Page SEO Audit

This file defines Beyond SEO's on-page SEO audit process.

On-page SEO is not keyword stuffing. It is the alignment of page topic, search intent, structure, trust, internal links, and conversion.

---

## 1. Purpose

On-page SEO answers:

```text
Does each important page clearly target the right intent and give search engines/users enough reason to rank and convert?
```

---

## 4. Title Tag Audit

Check whether the title:

```text
Clearly describes the page
Matches search intent
Includes primary topic naturally
Includes location if local
Is unique
Is click-worthy without being spammy
Does not repeat the same pattern across all pages
Does not overstuff keywords
```

Do not obsess over exact character count. Focus on clarity, uniqueness, and CTR.

Examples:

Weak:

```text
Services | Home
Dental Services
Best Best Best Dental Implants Arlington Heights Dentist
```

Strong:

```text
Dental Implants in Arlington Heights, IL | Dr. M. Shoaib Khan
AI Automation Course in Pakistan | BeyondTahir Academy
Emergency Dentist in Arlington Heights | Same-Day Dental Care
```

---

## 5. Meta Description Audit

Check whether the meta description:

```text
Matches page intent
Explains value
Includes service/location naturally
Has a CTA
Is unique
Avoids generic fluff
```

Good:

```text
Get dental implants in Arlington Heights with Dr. M. Shoaib Khan. Learn about candidacy, cost, recovery, and financing. Book a consultation today.
```

Weak:

```text
Welcome to our website. We provide the best services and high quality solutions for everyone.
```

---

## 6. H1 Audit

Check:

```text
One clear H1 for the main page topic
H1 matches page intent
H1 is visible
H1 is not just logo text
H1 is not repeated across many pages
H1 is not used for multiple CTA/design blocks
```

A page can technically have multiple H1s in modern HTML, but for SEO clarity and client implementation, Beyond SEO should recommend one clear main H1 per page unless the template has a justified structure.

---

## 7. Heading Structure Audit

Check H2/H3 sections for:

```text
Logical structure
Coverage of search intent
Process/cost/FAQ/trust sections
No skipped/confusing hierarchy where it hurts readability
No headings used only for design without meaning
```

Useful service-page H2s:

```text
What is [service]?
Who is a good candidate?
How the process works
Cost or financing
Benefits and risks
Why choose [provider]
FAQs
Book consultation
```

Useful course-page H2s:

```text
What you will learn
Who this course is for
Tools covered
Projects you will build
Curriculum
Certificate
FAQs
Enroll now
```

---

## 8. Search Intent Match

Classify intent:

```text
Transactional
Commercial investigation
Local service
Emergency/urgent
Informational
Comparison
Cost/pricing
Brand
Review/trust
Course/enrollment
AEO/question intent
```

Ask:

```text
Does this page satisfy the dominant intent?
Does SERP show service pages, blogs, directories, videos, maps, or product pages?
Is the current page the right format?
```

Example:

```text
If SERP for “family dentist arlington heights” ranks dedicated local service pages, a generic homepage is usually not enough.
```

---

## 9. Keyword Usage Rules

Use keywords naturally.

Check:

```text
Primary keyword in title/H1/body where appropriate
Secondary keywords covered in sections
Synonyms/entities included naturally
No forced density
No repeated exact-match stuffing
No unrelated keyword targeting
```

Modern keyword coverage means:

```text
Cover the topic completely enough that both users and search systems understand it.
```

## 11. Content Depth Audit

Not every page needs 2,000 words.

But high-intent money pages usually need enough detail to answer buying concerns.

Check:

```text
Does the page explain the service/course/product?
Does it answer cost/process/risk questions?
Does it show proof?
Does it compare options?
Does it explain next step?
Does it answer common objections?
```

Thin page signals:

```text
Under 400-500 words for competitive service intent
No FAQs
No proof
No process
No pricing/cost guidance where relevant
No internal links
No unique value
Copied/generic text
```

---

## 12. Internal Link Audit

Check:

```text
Homepage links to money pages
Service hub links to service pages
Support articles link to money pages
Money pages link to related services
Breadcrumbs exist where helpful
Anchor text is natural
Important pages are not orphaned
```

Good anchors:

```text
learn more about dental implants
AI automation course
emergency dental care in Arlington Heights
```

Weak anchors:

```text
click here
best dentist arlington heights best dentist arlington heights
```

---

## 13. Image and Media On-Page Audit

Check:

```text
Relevant images
Compressed images
Descriptive alt text
Before/after proof where compliant
Course/project screenshots where relevant
Videos with context/transcript if important
No giant hero media hurting LCP
```

---

## 14. CTA and Conversion Alignment

On-page SEO must support conversion.

Check:

```text
CTA above the fold
CTA repeated after key sections
Mobile CTA
Service-specific CTA
Trust proof near CTA
Short form
Phone/WhatsApp/booking where relevant
Clear next step
```

Weak CTA:

```text
Submit
Learn More
```

Better CTA:

```text
Book Dental Implant Consultation
Join AI Automation Course
Request Free SEO Audit
```

---

## 15. Trust Proof Audit

Check whether the page includes:

```text
Reviews
Testimonials
Case studies
Certifications
Expert credentials
Years of experience
Portfolio/project proof
Media mentions
Association memberships
Before/after results where compliant
Doctor/instructor/founder profile
```

For YMYL, trust proof is not optional.

---

## 16. On-Page Issue Severity

Use:

```text
Critical: page targets a money keyword but has wrong/no indexable content.
High: page exists but is too thin or intent mismatch.
Medium: metadata/headings/internal links need improvement.
Low: polish issues.
```

## 19. Common Mistakes

Do not:

```text
Recommend keyword stuffing.
Focus only on title length.
Ignore search intent.
Use same title/meta formula everywhere.
Ignore CTAs.
Ignore trust proof.
Ignore internal links.
Write blogs without connecting to money pages.
```

---

## 20. Final On-Page Gate

A page is on-page strong when:

```text
The topic is clear.
The intent is matched.
The structure is logical.
The content is useful and specific.
The page has trust proof.
The page links to/from related pages.
The page has a clear CTA.
The page can support AEO/GEO answers.
```
