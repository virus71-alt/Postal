# Full Site Audit Workflow

This file defines the complete Beyond SEO audit process for full website audits, SEO + AEO/GEO reviews, client reports, proposal reports, and 30/60/90-day growth plans.

A full-site audit is not a checklist dump. It is a decision system that identifies what is blocking ranking, visibility, authority, conversion, and AI-search readiness.

---

## 1. When to Use

Use this file when the user asks for:

```text
Full SEO audit
Website SEO audit
SEO + AEO audit
Technical + content audit
Client SEO report
90-day SEO plan
500-query growth plan
Website growth strategy
Competitor-based SEO plan
```

---

## 2. Core Questions

A Beyond SEO audit must answer:

```text
What is stopping this website from ranking?
What is stopping this website from converting?
What is stopping this website from becoming an authority?
What is stopping this website from appearing in AI/answer engines?
What should be fixed first?
What should be built next?
What data is missing?
```

Every issue must connect to at least one outcome:

```text
Ranking
Indexing
Crawlability
Topical authority
Local visibility
Backlink authority
AEO/GEO visibility
Conversion
Measurement
Revenue/query target
Risk reduction
```

---

## 3. Required Audit Sequence

Run the full audit in this order:

```text
1. Capability detection
2. Audit mode selection
3. Business and goal understanding
4. Site discovery
5. Crawlability and indexability
6. Page inventory
7. Technical SEO
8. On-page SEO
9. Content quality and E-E-A-T
10. Schema and structured data
11. Internal linking
12. Keyword and intent gap
13. Competitor gap
14. Local SEO if relevant
15. Backlink/authority gap
16. AEO/GEO readiness
17. Conversion SEO
18. Measurement and tracking
19. Issue prioritization
20. 30/60/90-day plan
```

Never jump to recommendations before choosing the audit mode and checking available data.

## 9. Crawlability and Indexability Review

Answer:

```text
Can search engines discover important pages?
Can search engines crawl them?
Can search engines index them?
Can search engines understand them?
Are important pages blocked, canonicalized away, empty, duplicated, or hidden behind JavaScript?
```

Critical examples:

```text
Important pages noindexed
Important pages blocked by robots.txt
Sitemap missing important pages
Canonical points to wrong URL
JavaScript renders no meaningful source content
Duplicate title/meta across many routes
Thin pages indexed at scale
Staging/dev pages indexed
Broken money pages
```

## 20. Issue Prioritization

Use this scoring model:

```text
Impact: 1-5
Effort: 1-5
Confidence: 1-5
Risk: 1-5
Priority Score = Impact + Confidence - Effort - Risk
```

Priority labels:

```text
Must Fix Now
High Impact Next
Strategic Build
Monitor
Ignore for Now
```

## 22. SEO Health Score

If scoring is requested, score out of 100:

```text
Technical SEO: 20
On-page SEO: 15
Content/E-E-A-T: 20
Keyword architecture: 15
Authority/backlinks: 10
Local SEO if relevant: 10
AEO/GEO readiness: 5
Conversion/tracking: 5
```

If local SEO is not relevant, redistribute local points into content, authority, and conversion.

Do not over-focus on the number. The explanation matters more.

---

## 23. Common Full-Audit Mistakes

Avoid:

```text
Listing 100 issues without priority.
Ignoring business goals.
Ignoring conversion.
Ignoring local SEO for local businesses.
Ignoring JavaScript rendering.
Ignoring E-E-A-T for YMYL.
Treating all pages equally.
Recommending blogs without mapping them to money pages.
Saying backlinks are needed without explaining which links and why.
Inventing search volume/rankings.
```

---

## 24. Final Quality Gate

Before finalizing, confirm:

```text
Did I choose the correct audit mode?
Did I list data sources?
Did I label missing data?
Did I separate confirmed from estimated?
Did I prioritize issues?
Did I connect recommendations to business impact?
Did I include AEO/GEO when relevant?
Did I include conversion when goals involve leads/queries?
Did I avoid fake guarantees?
```

If yes, deliver the report.
