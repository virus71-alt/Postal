# R2.md — How to upload PostalAtlas to the live server (from any computer)

_This is your "set it up again from scratch" guide. Follow it on a new PC, after
an OS reinstall, or whenever the one-click deploy stops working._

The website's files live in **Cloudflare R2** (a cloud storage bucket named
`postalatlas`). "Deploying" = copying your finished `server/` folder up to that
bucket. The tool that does the copying is called **rclone**.

---

## What you need (3 things)

1. **The finished website files** — the `server/` folder (or `out/` folder) that
   I generate. Without a fresh build, you can re-deploy whatever `server/` you
   already have.
2. **rclone** — the free upload tool.
3. **Your R2 keys** — 3 values that let rclone log in to your Cloudflare storage:
   - **Account ID:** `9106ad535e6a4c4539f5558b7cba714e`
   - **Access Key ID:** (secret — see "Getting your keys" below)
   - **Secret Access Key:** (secret — see below)
   - **Endpoint** (built from the account id):
     `https://9106ad535e6a4c4539f5558b7cba714e.r2.cloudflarestorage.com`
   - **Bucket name:** `postalatlas`

---

## Getting your keys

**Easiest:** if you still have the old machine, copy the file
`C:\prjt\Po\rclone-r2.conf` to the new one — it already has all the keys. Done.

**If you don't have that file**, make fresh keys:
1. Go to **dash.cloudflare.com → R2 → Manage R2 API Tokens → Create API Token**.
2. Permissions: **Object Read & Write**, scoped to the `postalatlas` bucket.
3. Create it and copy the **Access Key ID** and **Secret Access Key** (the secret
   is shown only once).

---

## Setup on WINDOWS (step by step)

1. **Install rclone:** open PowerShell and run:
   ```
   winget install Rclone.Rclone
   ```
   (or download from https://rclone.org/downloads/ and unzip it).

2. **Create the config file.** Make a file named `rclone-r2.conf` (anywhere easy,
   e.g. next to your `server` folder) with this content — paste your real keys in
   the two `...` spots:
   ```
   [r2]
   type = s3
   provider = Cloudflare
   access_key_id = YOUR_ACCESS_KEY_ID
   secret_access_key = YOUR_SECRET_ACCESS_KEY
   endpoint = https://9106ad535e6a4c4539f5558b7cba714e.r2.cloudflarestorage.com
   region = auto
   acl = private
   no_check_bucket = true
   ```

3. **Upload.** In PowerShell, from wherever your files are:
   ```
   rclone --config "rclone-r2.conf" sync "server" r2:postalatlas --transfers 48 --checkers 24 --progress
   ```
   - `sync` makes the bucket exactly match your `server` folder (uploads new/changed
     files, deletes removed ones).
   - It only sends what changed, so repeat deploys are fast.

4. Done — the site updates at **https://postalatlas.com** within seconds.

> The included `deploy-to-r2.cmd` just runs step 3 for you. If it works,
> double-clicking it is all you need.

---

## Setup on MAC or LINUX

1. **Install rclone:**
   ```
   curl https://rclone.org/install.sh | sudo bash
   ```
2. **Create `rclone-r2.conf`** with the exact same content shown above (your keys).
3. **Upload:**
   ```
   rclone --config ./rclone-r2.conf sync ./server r2:postalatlas --transfers 48 --checkers 24 --progress
   ```

## Quick checks / troubleshooting

- **"didn't find section in config file (r2)"** → rclone can't find your config.
  Use the full path in `--config`, e.g. `--config "C:\path\to\rclone-r2.conf"`.
- **"AccessDenied" / 403** → keys are wrong or the token lacks Object Read & Write,
  or it's scoped to a different bucket. Make a new token (see "Getting your keys").
- **Check the connection without uploading:**
  ```
  rclone --config "rclone-r2.conf" size r2:postalatlas
  ```
  This should print the number of files currently live (e.g. "65,536 objects").
- **See what would change without doing it:** add `--dry-run` to the sync command.

## IMPORTANT: don't try to deploy via Cloudflare Pages / "Compute" upload

The site has tens of thousands of files. Cloudflare Pages (the drag-and-drop
deploy) **rejects anything over 20,000 files**. That's exactly why we use R2 +
the Worker. **Always deploy with rclone to R2** (the steps above). The domain is
already wired to the Worker, which reads from this bucket — so an R2 upload is all
it takes to go live.

---

## What serves the site (so you understand the chain)

```
postalatlas.com  →  Cloudflare Worker named "postalatlas"  →  reads files from  →  R2 bucket "postalatlas"
```

You normally never touch the Worker — it's already deployed and just works. You
only ever update the **files in the bucket** (that's what these steps do). If the
Worker ever needs redeploying, that's a separate tool (`wrangler`) — ask me.

---

## Keep these safe

- `rclone-r2.conf` contains your secret key — **never** put it on GitHub or share
  it. (It's already in `.gitignore`.)
- If a key ever leaks, delete that token in the Cloudflare R2 dashboard and make a
  new one, then update `rclone-r2.conf`.
